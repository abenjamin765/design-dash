Wireframe Component Library

Monochrome lo-fi patterns for Design Dash HTML wireframes. Use these classes before writing new CSS. Components and rule notes are sourced from the ui-design skill's interaction decision tree (component_catalog + rules).

Principles

  1. Choose components from the user's goal first.
  2. Treat numeric thresholds as defaults; contextual overrides outrank counts.
  3. Keep essential information and consequential choices visible.
  4. Prefer native semantic elements.
  5. Do not turn read-only tabular information into an interactive grid.
  6. Do not hide primary actions or task-critical help.
  7. Insert review and confirmation for consequential AI actions.
