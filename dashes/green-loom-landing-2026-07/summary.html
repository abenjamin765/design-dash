Green Loom Landing Page — PLAN

Dash: green-loom-landing-2026-07 Date: 2026-07-10 Tier: High-stakes Concept: B-trimmed "Guided Depth"

Problem

Green Loom has no shipped public presence. Prospective operators and agencies can't discover or evaluate it, and no early-access demand signal exists. This plan ships a WordPress-delivered landing page with brand identity v0 that converts visitors into early-access signups (25 in 90 days) and discovery calls — while staying honest about pre-launch stage.

Primary user

Prospective operator (dispensary owner/operator); secondary: agency decision-maker; tertiary: founder-maintainer.

Chosen concept

B-trimmed: Guided Depth — 8-section narrative spine (merged from the 11-section draft), form vendor + scheduler, founder-note proof block (borrowed from Concept C). Runner-up: A "One Promise" (revisit if underconverting). Full scoring: requirements.md §3.

Scenarios

S1 — Curious operator (happy path → Lead)

Hero (plain-terms answer) → problem → pillars → optional module expand → closing CTA → signup success.

S2 — Skeptical agency (→ Discovery Call)

Hero → FAQ ("Is this a POS?") → suite statuses → schedule call.

S3 — Direct intent (→ Lead)

Sticky skip-to-signup anchor → form.

S4 — Founder maintenance

WP block editor edits; Brand Tokens keep styling consistent; no code required.

Key decisions

  • WordPress delivery — dogfoods GL Storefront/Studio direction
  • One P per context: Join early access owns primary; call CTA is secondary everywhere
  • Module status badges always visible (anti-overpromise, canonical rule)
  • Founder story + domain depth replaces fabricated social proof
  • Email-only minimum form; role optional (guardrail M-003 via optional field)
  • FAQ items are section content, not objects; "Landing Page" is the system, not an object
