# Canonical examples

Examples should help a designer understand both the method and the expected quality of its artifacts. Each canonical example should include:

- a short project brief and computed tier;
- evidence excerpts with provenance and confidence labels;
- an assumption register showing at least one status change;
- user and system mental models with a documented reconciliation;
- object model, flow, concepts, and selection rationale;
- wireframes covering happy, empty, loading, error, permission, and at-scale states where applicable;
- accessibility and ethics/equity findings, plus privacy review when applicable;
- final requirements, summary, acceptance criteria, and learning plan.

Prefer small, realistic examples over polished fiction. Redact personal information and clearly label synthetic evidence. Future examples should be checked against `method/method.yaml` so tool-specific output does not quietly redefine the method.
